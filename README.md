# VoxLens — AI-Powered News Platform 🔍

VoxLens is a full-stack AI-powered news platform that makes news easier to understand, more accessible, and more engaging. It features smart summaries, impact analysis, text-to-speech, quizzes, timelines, discussions, and an AI chatbot.

---

## ✨ Features

| Feature | Description |
|---------|-------------|
| 📍 **Location-Based News** | Auto-detects city via geolocation and serves local news |
| 🔊 **Read News Aloud** | TTS for accessibility — OpenAI TTS or browser fallback |
| 💡 **So What?** | AI explains why news matters, citizen impact, future implications |
| 📋 **Smart Brief** | 5-bullet AI summary of any article |
| 🧠 **Learning Quiz** | Multiple-choice comprehension quiz for each article |
| 🕐 **Story Timeline** | Visual timeline of how a story evolved |
| ⚖️ **Pros & Cons** | Balanced viewpoints generated by AI |
| 🔍 **News Search** | Search by keyword or paste article URL for analysis |
| 💬 **Community Discussion** | Comment on articles with moderation |
| 🤖 **AI Chatbot (VoxBot)** | Ask questions about news — context-aware AI assistant |

## 🚀 Local Setup

### Prerequisites
- Python 3.9+
- pip

### Step 1: Clone and configure

```bash
git clone https://github.com/youruser/voxlens.git
cd voxlens

# Copy environment template
cp .env.example .env
```

### Step 2: Add API keys to `.env`

```env
NEWS_API_KEY=your_newsapi_key_here
OPENAI_API_KEY=your_openai_api_key_here
```

**Get your keys:**
- **NewsAPI** (free): https://newsapi.org/register
- **OpenAI** (paid, optional): https://platform.openai.com/api-keys

> **Note:** VoxLens works without OpenAI — it uses intelligent fallback responses for all AI features. Without NewsAPI, it uses built-in demo articles.

### Step 3: Install backend dependencies

```bash
cd backend
pip install -r requirements.txt
```

### Step 4: Start the backend

```bash
# From the backend/ directory
python app.py

# Or with uvicorn directly:
uvicorn app:app --reload --port 8000
```

Backend runs at: `http://localhost:8000`
API docs at: `http://localhost:8000/docs`

### Step 5: Open the frontend

Open `frontend/index.html` in your browser directly, **or** serve it:

```bash
# From the project root (simple HTTP server):
cd frontend
python -m http.server 3000
```

Open: `http://localhost:3000`

## ☁️ Deployment

### Deploy on Render (Recommended — Free Tier)

1. Push your code to GitHub
2. Go to [render.com](https://render.com) and create a **New Web Service**
3. Connect your GitHub repository
4. Configure:
   - **Root Directory**: `backend`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn app:app --host 0.0.0.0 --port $PORT`
   - **Environment**: Python 3
5. Add environment variables in Render dashboard:
   - `NEWS_API_KEY` = your NewsAPI key
   - `OPENAI_API_KEY` = your OpenAI key
6. Deploy!

For the frontend, deploy to **Render Static Site** or **Netlify**:
- Set `frontend/` as the publish directory
- Update `API_BASE` in `script.js` to your Render backend URL

### Deploy on Railway

1. Install Railway CLI: `npm install -g @railway/cli`
2. Login: `railway login`
3. From the `backend/` directory:
```bash
railway init
railway up
```
4. Add environment variables in Railway dashboard
5. For frontend: use Railway's static hosting or Netlify

## 🔑 API Endpoints Reference

### News
| Endpoint | Description |
|----------|-------------|
| `GET /api/news/headlines?country=in&category=general` | Top headlines |
| `GET /api/news/local?city=Mumbai` | City-specific news |
| `GET /api/news/search?query=climate` | Keyword search |
| `POST /api/news/from-url` | Fetch article from URL |

### AI Analysis
| Endpoint | Description |
|----------|-------------|
| `POST /api/ai/smart-brief` | 5-bullet summary |
| `POST /api/ai/so-what` | Impact analysis |
| `POST /api/ai/pros-cons` | Balanced viewpoints |
| `POST /api/ai/quiz` | Comprehension quiz |
| `POST /api/ai/timeline` | Story timeline |
| `POST /api/ai/sentiment` | Tone analysis |

### Other
| Endpoint | Description |
|----------|-------------|
| `POST /api/tts/speak` | Text-to-speech audio |
| `POST /api/chat` | Chatbot message |
| `GET /api/discussions?article_url=...` | Get comments |
| `POST /api/discussions/comment` | Post comment |
| `POST /api/discussions/like` | Like a comment |
